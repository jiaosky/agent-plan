# 第 12 章：算法题与面试验收

## 1. 学习什么

本章学习传统后端转 AI 应用后端时，算法题应该怎么准备。

这里先说结论：你不需要一个月内把算法刷成竞赛水平。你的目标是避免在基础题上被筛掉，并且能用 Go 稳定写出常见题型。

你需要掌握以下内容：

- AI 应用后端面试为什么仍然会考算法。
- 一个月内优先刷哪些题型。
- 哪些题型可以暂时放低优先级。
- Go 刷题时常见坑。
- 每种题型的识别信号和模板思路。
- 如何用算法题训练表达能力。
- 如何做每日训练计划。
- 如何验收自己是否准备到位。
- 算法题和真实 AI 后端工作的关系。

本章学完后，你应该能说清楚：

> 我不是准备算法竞赛，而是针对后端和 AI 应用岗准备高频基础题型：哈希、双指针、滑动窗口、栈队列、二分、DFS/BFS、基础动态规划，并用 Go 熟练实现。

## 2. 为什么学习

你想去大模型公司做 AI 应用后端，不是训练算法岗。但算法题仍然可能出现。

原因很现实：

- 面试流程可能统一使用算法题筛选。
- 后端岗位仍然考基础编码能力。
- Go 岗会看你是否能写出清晰、无 bug 的代码。
- AI 应用公司也有大量工程岗位，不等于不考基础。
- 算法题能快速暴露边界处理、复杂度意识、调试能力。

但你不能把一个月全部花在刷题上。

你的优先级应该是：

```text
AI 应用后端核心知识
  + 一个可讲项目
  + Go 场景表达
  + 高频算法题
```

算法题的目标：

- 简单题稳定 AC。
- 中等题常见套路能做。
- 面试时能讲思路、复杂度和边界。
- 不在数组、哈希、二分、DFS 这种基础题上卡死。

不要追求：

- 偏竞赛题。
- 复杂图论。
- 高级动态规划。
- 红黑树、线段树、网络流。
- 一个题刷半天钻牛角尖。

一个月内，你要做的是“够面试用”，不是成为算法专家。

## 3. 知识详细内容

### 3.2 刷题的正确目标

不要只看“做过多少题”。

更重要的是：

- 看到题能识别题型。
- 能说出为什么用这个方法。
- 能写出 Go 代码。
- 能处理边界。
- 能分析时间复杂度。
- 能复盘错因。

每道题复盘时问自己：

```text
这题属于什么类型？
关键条件是什么？
暴力解是什么？
优化点在哪里？
时间复杂度是多少？
边界用例有哪些？
Go 实现有什么坑？
```

### 3.3 哈希表

识别信号：

- 查找是否存在。
- 统计频次。
- 两数之和。
- 去重。
- 字母异位词。
- 前缀和 + 出现次数。

典型题：

- Two Sum。
- 字母异位词分组。
- 最长连续序列。
- 和为 K 的子数组。

Go 注意点：

```go
mp := make(map[int]int)

if v, ok := mp[key]; ok {
    _ = v
}
```

面试表达：

> 看到需要快速判断元素是否出现或统计频次，我会优先考虑哈希表，用空间换时间，把查找从 O(n) 降到 O(1) 均摊。

### 3.4 双指针

识别信号：

- 有序数组。
- 原地去重。
- 两端向中间靠。
- 快慢指针。
- 链表环。

典型题：

- 有序数组两数之和。
- 删除有序数组重复项。
- 盛最多水的容器。
- 判断链表是否有环。
- 反转字符串。

常见模式：

```text
left 从左
right 从右
根据条件移动其中一个指针
```

面试表达：

> 双指针适合利用顺序或两个位置之间的关系，减少嵌套循环。

### 3.5 滑动窗口

识别信号：

- 连续子数组。
- 连续子串。
- 最长/最短窗口。
- 窗口内满足某个条件。

典型题：

- 无重复字符的最长子串。
- 最小覆盖子串。
- 长度最小的子数组。
- 找到字符串中所有字母异位词。

模板：

```text
right 扩大窗口
更新窗口状态
while 窗口不满足条件:
  left 收缩窗口
更新答案
```

Go 注意：

- 字符串按 byte 还是 rune 处理要看题目。
- 英文字母可以用 `[26]int`。
- Unicode 字符要用 `[]rune`。

面试表达：

> 滑动窗口适合连续区间问题，通过维护窗口状态，把枚举所有子数组的 O(n^2) 降到 O(n)。

### 3.6 栈和队列

栈识别信号：

- 括号匹配。
- 最近未匹配元素。
- 单调栈。
- 表达式。
- 路径简化。

典型题：

- 有效括号。
- 最小栈。
- 每日温度。
- 柱状图最大矩形。

队列识别信号：

- BFS。
- 层序遍历。
- 任务调度。

Go 实现栈：

```go
stack := []int{}
stack = append(stack, x)
x = stack[len(stack)-1]
stack = stack[:len(stack)-1]
```

Go 实现队列：

```go
queue := []int{}
queue = append(queue, x)
x = queue[0]
queue = queue[1:]
```

面试表达：

> 栈适合处理最近相关的问题，队列适合处理先进先出和 BFS 层序遍历。

### 3.7 二分查找

识别信号：

- 有序数组。
- 单调性。
- 查找边界。
- 最小可行值。
- 最大可行值。

典型题：

- 二分查找。
- 搜索插入位置。
- 搜索旋转排序数组。
- 在排序数组中查找元素第一个和最后一个位置。
- 求最小满足条件的值。

基础模板：

```go
left, right := 0, len(nums)-1
for left <= right {
    mid := left + (right-left)/2
    if nums[mid] == target {
        return mid
    } else if nums[mid] < target {
        left = mid + 1
    } else {
        right = mid - 1
    }
}
return -1
```

面试表达：

> 二分的关键不是数组有序，而是答案空间存在单调性。边界题要明确查找的是等于值、左边界还是右边界。

### 3.8 DFS / BFS

DFS 识别信号：

- 递归遍历。
- 树。
- 图。
- 岛屿问题。
- 回溯搜索。

BFS 识别信号：

- 最短步数。
- 层序遍历。
- 多源扩散。

典型题：

- 二叉树最大深度。
- 二叉树层序遍历。
- 岛屿数量。
- 腐烂的橘子。
- 单词接龙基础。

DFS 模板：

```go
func dfs(i, j int) {
    if outOfBounds || visited {
        return
    }
    visited = true
    for each direction {
        dfs(ni, nj)
    }
}
```

BFS 模板：

```go
queue := []Node{start}
visited[start] = true
for len(queue) > 0 {
    cur := queue[0]
    queue = queue[1:]
    for _, next := range neighbors(cur) {
        if !visited[next] {
            visited[next] = true
            queue = append(queue, next)
        }
    }
}
```

面试表达：

> DFS 适合遍历所有可能路径或连通块，BFS 适合找最短步数和层级扩散。

### 3.9 链表基础

识别信号：

- 指针操作。
- 反转。
- 合并。
- 快慢指针。
- 删除节点。

典型题：

- 反转链表。
- 合并两个有序链表。
- 链表是否有环。
- 删除倒数第 N 个节点。
- 相交链表。

Go 注意：

- 使用 dummy node 简化头节点变化。
- 注意 nil 判断。
- 反转链表要保存 next。

面试表达：

> 链表题重点是指针顺序和边界，dummy 节点可以减少删除或合并时的特殊判断。

### 3.10 简单动态规划

一个月内只准备基础 DP。

识别信号：

- 最优值。
- 方案数。
- 当前状态依赖前面状态。
- 子问题重复。

典型题：

- 爬楼梯。
- 打家劫舍。
- 最大子数组和。
- 零钱兑换基础。
- 最长递增子序列基础。

DP 思路：

```text
定义 dp[i] 表示什么
找状态转移
确定初始值
确定遍历顺序
返回结果
```

例如爬楼梯：

```text
dp[i] = dp[i-1] + dp[i-2]
```

面试表达：

> DP 的关键是定义状态和状态转移。一个月内我优先掌握线性 DP 和基础背包，不追复杂区间 DP 或树形 DP。

### 3.11 Go 刷题常见坑

#### slice 共享底层数组

回溯题里要拷贝结果：

```go
tmp := append([]int(nil), path...)
res = append(res, tmp)
```

#### map 默认零值

判断存在要用：

```go
v, ok := mp[key]
```

不要把不存在和存在但值为 0 混淆。

#### 字符串 byte / rune

英文题用 byte 通常可以。

中文或 Unicode 要用 rune：

```go
runes := []rune(s)
```

#### 整数溢出

二分 mid：

```go
mid := left + (right-left)/2
```

#### 队列切片内存

刷题可以用 `queue = queue[1:]`，生产代码要注意长队列底层数组引用。

### 3.13 面试答题流程

算法题面试时，不要一上来闷头写。

流程：

```text
确认题意
举例
说暴力解
提出优化
说明复杂度
写代码
跑样例
检查边界
```

口述示例：

> 这题要求连续子数组，并且要找最长长度，我会先考虑滑动窗口。右指针扩张窗口，窗口不满足条件时移动左指针，过程中更新答案。每个元素最多进出窗口一次，所以时间复杂度是 O(n)。

### 3.14 常见误区

误区一：刷题越多越好。

不对。不会复盘，刷再多也只是眼熟。

误区二：只背答案。

面试官改一点条件就会崩。

误区三：只刷困难题。

一个月内更应该保证简单和常见中等题稳定。

误区四：忽视 Go 语法细节。

slice、map、字符串、nil 指针都容易出 bug。

误区五：算法和工作完全无关。

算法题不是直接等于工作，但能体现编码基本功。面试里它仍然是门槛。

### 4.2 面试问答题

问题一：你算法准备到什么程度？

参考回答：

> 我不是按算法岗深度准备，而是针对后端和 AI 应用岗准备高频基础题型，包括数组、哈希、双指针、滑动窗口、栈队列、二分、链表、DFS/BFS 和基础动态规划。目标是简单题稳定，中等题常见套路能做，并能用 Go 讲清复杂度和边界。

问题二：你怎么判断一道题用滑动窗口？

参考回答：

> 如果题目要求连续子数组或子串，并且目标是最长、最短或满足某个窗口条件，我会考虑滑动窗口。通过右指针扩张、左指针收缩维护窗口状态，通常可以把 O(n^2) 优化到 O(n)。

问题三：二分查找什么时候能用？

参考回答：

> 不只是数组有序才能用，关键是搜索空间有单调性。比如查找某个值、查找边界、或者找最小满足条件的答案，都可以考虑二分。

问题四：DFS 和 BFS 怎么选？

参考回答：

> DFS 适合遍历所有路径、连通块、树的递归问题；BFS 适合层序遍历和最短步数问题。如果题目问最少几步，一般优先考虑 BFS。

## 6. 演变过程与后续方向

### 6.1 从传统后端算法面试到 AI 应用岗

传统后端面试常考算法，是为了筛编码基本功。

AI 应用岗虽然更关注大模型工程，但仍然可能沿用算法筛选。

所以你不能完全不准备。

### 6.2 从刷题数量到题型掌握

早期很多人追求刷题数量。

更有效的方式是按题型掌握：

```text
看到题 -> 识别类型 -> 套合适思路 -> 处理边界
```

题型意识比题量更重要。

### 6.3 从写代码到讲代码

面试不是在线判题。

你还要能讲：

- 为什么这样做。
- 复杂度是多少。
- 边界是什么。
- 有没有其他解法。
- 为什么不用另一个方法。

### 6.4 后续方向

如果你后面时间更多，可以继续补：

- 堆。
- 并查集。
- 拓扑排序。
- 前缀和进阶。
- 单调栈。
- 更系统的动态规划。
- 图论基础。

但这不是第一个月的主线。

## 7. 工作中典型场景

### 场景一：RAG 合并召回结果

向量检索和关键词检索各返回一批 chunk，需要：

- 去重。
- 按分数排序。
- 取 top_k。

这会用到哈希、排序、堆。

### 场景二：Agent 步骤执行

Agent 任务可以抽象成步骤队列或状态图。

你需要理解：

- 队列。
