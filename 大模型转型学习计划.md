# 大模型公司转型学习计划

## 目标定位

一个月后的目标不是成为算法工程师，而是成为一个能把大模型能力落地到业务系统里的后端工程师。

面试时可以这样介绍自己：

> 我之前主要做传统后端，现在重点补 AI 应用后端。我关注的不是模型训练，而是如何把大模型稳定接入业务系统，包括 RAG、SSE 流式输出、Tool Calling、MCP、文件权限、安全隔离、限流重试和日志观测。我用 Go 做过一个知识库问答 Demo，可以讲完整链路。

## 学习主线

只抓三件事：

1. 会讲大模型应用后端。
2. 做一个能展示的小项目。
3. 补 Go 和算法面试基础。

## 面试必须掌握

### 1. 大模型应用基本范式

需要掌握：

- Prompt：如何组织模型输入。
- Token：模型输入输出的计量单位。
- RAG：检索增强生成，把企业文档或知识库提供给模型。
- Agent：模型根据目标调用工具完成任务。
- Tool Calling / Function Calling：模型决定调用哪个函数或工具。
- MCP：模型连接外部工具、文件、数据库、浏览器等能力的标准协议。
- Embedding：把文本转成向量，用于相似度搜索。
- Vector DB：向量数据库，如 Milvus、pgvector、Qdrant。

需要能说清楚：

```text
用户问题 -> 意图理解 -> 检索或工具调用 -> 拼 Prompt -> 模型生成 -> 结果校验 -> 返回用户
```

### 2. 后端在大模型公司做什么

典型工作：

- 接 OpenAI、Claude、通义、DeepSeek、私有模型 API。
- 做对话系统、流式输出、会话管理。
- 做 RAG 知识库、文件上传、切片、向量检索。
- 做 Agent 工具调用、MCP Server。
- 做权限、审计、计费、限流、任务队列。
- 做模型调用日志、成本统计、失败重试。
- 做沙盒执行代码、文件管理、安全隔离。

### 3. MCP / SSE / Channel / 沙盒

需要掌握到能解释：

- MCP：大模型连接外部工具和数据源的一套标准协议。
- SSE：服务端持续向前端推送数据，常用于大模型流式输出。
- Channel：消息通道、事件通道或工具调用通道，具体含义看项目上下文。
- 沙盒：限制模型执行代码或命令的权限，避免危险操作。
- 文件管理：上传、解析、分块、索引、权限、版本、引用来源。

### 4. Go 高频场景

需要准备这些表达：

- goroutine + channel：异步任务、并发模型调用、流式消息转发。
- context：超时取消、请求链路控制。
- sync.WaitGroup / errgroup：并发执行多个任务。
- interface：抽象不同模型供应商。
- http server/client：API 网关、模型代理、SSE 输出。
- Redis：限流、会话缓存、任务状态。
- MQ：文件解析、Embedding、异步生成。

面试表达：

> Go 适合做模型网关、SSE 流式服务、异步任务调度、RAG 后端、工具调用编排，因为它并发模型简单，部署成本低，服务稳定。

## 工作必须掌握

### 1. 模型调用工程化

- token 计算。
- 超时重试。
- 限流。
- fallback 模型。
- 流式输出。
- 模型调用日志。
- 成本统计。

### 2. RAG 工程

- 文件上传。
- PDF / Word / Markdown 解析。
- 文本切片。
- Embedding。
- 向量检索。
- rerank 重排。
- 引用来源。
- 权限隔离。

### 3. Agent / Tool Calling

- 模型什么时候调用工具。
- 工具参数怎么校验。
- 工具失败怎么处理。
- 多轮调用怎么保存状态。
- 如何避免模型乱调用危险工具。

### 4. 安全

- Prompt Injection。
- 越权访问文件。
- 沙盒命令限制。
- API Key 管理。
- 用户隔离。
- 审计日志。

### 5. 日常沟通关键词

需要熟悉这些说法：

- 这个需求是走 RAG 还是 Tool Calling？
- 是否需要流式返回？
- 是否要保留引用来源？
- 工具调用失败是否重试？
- 这个 Agent 是否需要状态机？
- 是否有 token 成本预算？
- 文件权限是否跟用户或租户隔离？

## 30 天计划

## 第 1 周：建立概念

目标：听懂别人说什么，面试不露怯。

每天 1.5 到 2 小时。

### 第 1 天：大模型应用整体链路

学习内容：

- Prompt 是什么。
- Token 是什么。
- 模型 API 怎么调用。
- 普通聊天和业务系统接入有什么区别。

需要能说：

> 大模型应用不是直接把用户问题丢给模型，而是要做上下文构造、权限控制、工具调用、结果校验和日志记录。

### 第 2 天：RAG

学习内容：

- 文档上传。
- 文本切片。
- Embedding。
- 向量库。
- 相似度检索。
- 拼 Prompt。
- 引用来源。

流程：

```text
文件上传 -> 解析文本 -> 切片 -> Embedding -> 入向量库
用户提问 -> 向量检索 -> 拼上下文 -> 调模型 -> 返回答案
```

### 第 3 天：SSE 流式输出

学习内容：

- 为什么大模型要流式返回。
- SSE 和 WebSocket 区别。
- Go 怎么实现 SSE。
- 前端如何增量显示模型回复。

需要能说：

> 大模型生成时间长，后端一般用 SSE 把 token 增量推给前端，体验比等待完整结果更好。

### 第 4 天：Tool Calling / Function Calling

学习内容：

- 模型什么时候调用工具。
- 工具参数怎么定义。
- 工具失败怎么办。
- 工具结果怎么再交给模型总结。

流程：

```text
用户问题 -> 模型判断需要工具 -> 后端执行工具 -> 工具结果返回模型 -> 模型生成最终答案
```

### 第 5 天：MCP

学习内容：

- MCP 是什么。
- MCP Client / Server 是什么。
- Tools / Resources / Prompts 是什么。
- 它和普通 API 调用有什么区别。

需要能说：

> MCP 可以理解成大模型连接外部工具和数据源的一套标准协议，方便模型统一调用文件、数据库、浏览器、内部系统等能力。

### 第 6 天：沙盒和文件管理

学习内容：

- 为什么模型执行命令危险。
- 文件上传后的权限问题。
- 用户隔离、租户隔离。
- 代码执行沙盒。
- 操作审计。

需要能说：

> 只要让模型碰文件、命令、数据库，就必须考虑权限边界和沙盒隔离。

### 第 7 天：复盘整理

产出一份小笔记：

```text
AI 后端必懂概念.md
```

内容只写：

- RAG 是什么。
- SSE 是什么。
- Tool Calling 是什么。
- MCP 是什么。
- 沙盒为什么重要。
- AI 后端日常做什么。

## 第 3 周：补 Go 面试和 AI 后端设计

目标：把“我会用 Go”变成“我知道 Go 在 AI 后端怎么用”。

### 第 15 天：Go 并发

准备：

- goroutine。
- channel。
- WaitGroup。
- context。

AI 场景表达：

> 可以并发调用多个检索源，或者并发执行多个工具，再把结果汇总给模型。

### 第 16 天：context 和超时控制

重点：

- 请求超时。
- 用户取消。
- 模型调用超时。
- 下游服务超时。

面试表达：

> 模型调用可能很慢，所以每次请求都应该带 context，避免请求堆积。

### 第 17 天：接口抽象

准备：

```go
type LLMClient interface {
    Chat(ctx context.Context, req ChatRequest) (*ChatResponse, error)
    Stream(ctx context.Context, req ChatRequest) (<-chan ChatChunk, error)
}
```

需要能说：

> 这样可以兼容 OpenAI、DeepSeek、通义、私有模型，不把业务代码绑死在某个供应商上。

### 第 18 天：限流、重试、降级

学习：

- Redis 限流。
- 模型失败重试。
- fallback 模型。
- 请求队列。
- 成本控制。

面试表达：

> 模型接口贵且不稳定，必须做限流、超时、重试和日志。

### 第 19 天：RAG 系统设计

准备问题：

> 如果让你设计一个企业知识库问答系统，你怎么做？

回答结构：

```text
文件上传 -> 解析 -> 切片 -> 向量化 -> 入库
用户提问 -> 检索 -> rerank -> 拼 Prompt -> 调模型 -> 返回答案和引用
```

### 第 20 天：Agent / MCP 系统设计

准备问题：

> 如果让模型调用公司内部系统，你怎么设计？

回答结构：

```text
定义工具 -> 参数校验 -> 权限判断 -> 执行工具 -> 返回结果 -> 模型总结
```

重点强调：

- 不能让模型直接操作数据库。
- 工具要白名单。
- 参数要校验。
- 操作要审计。

### 第 21 天：模拟面试

录音回答：

- 你为什么从传统后端转 AI 后端？
- 你理解的 RAG 是什么？
- SSE 怎么实现？
- MCP 是什么？
- Go 在大模型应用里适合做什么？
- 如何保证模型调用稳定？
- 如何防止模型越权访问文件？
