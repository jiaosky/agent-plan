适合马上要面试、时间紧的情况。

```text
第 1 天：第01章 + 第02章
第 2 天：第03章 + 第04章
第 3 天：第05章 + 第06章
第 4 天：第07章 + 第08章
第 5 天：第09章 + 第10章
第 6 天：第11章 + 第12章
```

每章不要追求一次全背下来，先做到：

- 能讲出本章核心概念。
- 能回答本章 3 个面试题。
- 能说出 1 个工作场景。

## 面试优先复习

面试前优先背熟这些问题：

- AI 应用后端和传统后端有什么区别？
- RAG 是什么，难点在哪里？
- SSE 流式输出怎么实现？
- Tool Calling 如何保证安全？
- MCP 是什么，和 Tool Calling 有什么区别？
- Agent 后端怎么设计？
- 沙盒解决什么问题？
- Go 在 AI 后端里有哪些典型场景？
- 你的 Go AI Knowledge Assistant 项目怎么讲？
- 你的项目有什么不足，后续怎么演进？

对应章节：

```text
第01章：转型定位
第05章：RAG
第08章：MCP
第11章：Go 场景
第13章：项目
第14章：面试表达
```

## 每章怎么学

每章按这个顺序读：

1. 先读“学习什么”和“为什么学习”。
2. 再读“知识详细内容”。
3. 重点看“工作中典型场景”。
4. 最后用“考题、编程题与验收”自测。

每章完成标准：

- 我能用自己的话解释核心概念。
- 我能回答本章面试题。
- 我能说出这个知识在工作中的典型场景。
- 如果有编程题，我知道大概怎么写。

## 最重要的学习目标

不要把自己包装成算法训练专家。

你的定位应该是：

> 我是传统后端转 AI 应用后端，关注的是如何把大模型能力稳定、安全、可观测地接入业务系统。

最终你要能讲清：

```text
模型调用
Prompt
SSE
RAG
文件管理
Tool Calling
MCP
Agent
沙盒
Go 工程化
项目实战
面试表达
```

## 下一步建议

优先按第 13 章实现：

```text
Go AI Knowledge Assistant
```

最小可用版本只需要做到：

- 普通聊天接口。
- SSE 流式聊天接口。
- 文件上传。
- 文本切片。
- 简易关键词检索。
- RAG Prompt。
- 模型调用日志。

这个项目做出来后，再用第 14 章整理简历和面试话术。
