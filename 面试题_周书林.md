## 一、Java 基础能力（问题-答案-追问-答案）

### 1. HashMap 与并发安全

**面试题**：HashMap 的底层结构是什么？为什么说 HashMap 线程不安全？并发场景下用什么替代？

**参考答案要点：**
- **结构**：JDK8 是数组 + 链表/红黑树；hash 算桶下标，冲突用链地址法；链表长度>8 且数组≥64 转红黑树
- **不安全**：并发 put 可能丢数据、覆盖、甚至 JDK7 扩容形成环导致死循环
- **替代**：ConcurrentHashMap；或 Collections.synchronizedMap（性能较差）

**追问入口：**
- 若答出结构 → "put 时发生哈希冲突怎么处理？红黑树阈值是多少？"
- 若答得流畅 → "ConcurrentHashMap 在 JDK8 里是怎么保证线程安全的？"

**追问参考答案：**
- 冲突时挂到链表；链表>8 且数组≥64 转红黑树；退化为链表是 6，避免频繁转换
- CAS + synchronized 锁链表头/树根；锁粒度细，并发度高

### 2. synchronized 与 volatile

**面试题**：synchronized 和 ReentrantLock 有什么区别？volatile 的作用是什么？能保证原子性吗？

**参考答案要点：**
- **synchronized**：JVM 内置，自动加解锁；不可中断；非公平
- **ReentrantLock**：API 显式 lock/unlock；可中断、可 tryLock、可公平；需在 finally 里 unlock
- **volatile**：保证可见性、禁止指令重排；**不保证原子性**；i++ 用 volatile 仍不安全，需 synchronized 或 AtomicInteger

**追问入口：**
- 若答出 volatile → "i++ 用 volatile 修饰为什么还是不安全？"
- 若答得流畅 → "什么场景用 volatile？什么场景用 synchronized？"

**追问参考答案：**
- i++ 是读-改-写三步，非原子；多线程会丢更新
- volatile 适合状态标志位、单次读写；synchronized 适合复合操作、需要互斥的代码块

---

### 3. 线程池

**面试题**：线程池的核心参数有哪些？拒绝策略有哪几种？你们项目里线程池一般怎么配置？

**参考答案要点：**
- **核心参数**：corePoolSize、maximumPoolSize、workQueue、keepAliveTime、threadFactory、RejectedExecutionHandler
- **执行顺序**：核心未满则新建线程；满了入队列；队列满且未达最大线程则新建非核心；都满则拒绝策略
- **拒绝策略**：AbortPolicy 抛异常、CallerRunsPolicy 调用者执行、DiscardPolicy 丢弃、DiscardOldestPolicy 丢最老

**追问入口：**
- 若答出参数 → "核心线程数和最大线程数一般怎么设？IO 密集和 CPU 密集有区别吗？"
- 若答得流畅 → "线程池用完后需要 shutdown 吗？不关会怎样？"

**追问参考答案：**
- CPU 密集约等于核数；IO 密集可核数*(1+IO耗时/CPU耗时)，需压测调优
- 需要；不 shutdown 线程不会回收，线程池引用在的话 JVM 不退出，且可能造成泄漏

### 4. Spring IoC 与 AOP

**面试题**：Spring 的 IoC 和 AOP 分别解决什么问题？SSM 里 Spring、SpringMVC、MyBatis 各自负责什么？

**参考答案要点：**
- **IoC**：控制反转，对象的创建和依赖交给容器管理，解耦
- **AOP**：切面，把日志、事务等横切逻辑与业务分离
- **SSM**：Spring 是 IoC 容器、事务、AOP；SpringMVC 处理 HTTP、Controller、参数绑定；MyBatis 是 ORM、SQL 映射

**追问入口：**
- 若答出 SSM → "SpringBoot 和传统 SSM 比有哪些优势？"
- 若答得流畅 → "Spring 事务传播机制了解吗？REQUIRED 和 REQUIRES_NEW 有什么区别？"

**追问参考答案：**
- SpringBoot 自动配置、内嵌容器、starter 依赖、开发效率高；SSM 需手写配置多
- REQUIRED 加入当前事务；REQUIRES_NEW 挂起当前事务、新建独立事务，常用于日志等独立子事务

---

### 5. 异常与排查

**面试题**：什么情况下会 OOM？如何排查？线上接口变慢，你一般从哪些方面排查？

**参考答案要点：**
- **OOM**：堆溢出（对象过多、泄漏）、栈溢出（递归过深）、元空间（类过多）；-XX:+HeapDumpOnOutOfMemoryError 生成 dump，MAT 分析
- **接口变慢**：应用（jstack 看线程、死锁、CPU 热点）、DB（慢 SQL、explain、连接池）、Redis（slowlog、bigkey）、MQ（堆积）

**追问入口：**
- 若答出 OOM → "堆 dump 怎么分析？看哪些指标？"
- 若答得流畅 → "jstack 怎么用？怎么排查死锁？"

**追问参考答案：**
- 用 MAT 看堆中对象占比、GC Roots 引用链，找泄漏对象
- jstack <pid> 导出线程栈；搜索 "deadlock" 或 "Found one Java-level deadlock"；看锁的持有和等待关系
