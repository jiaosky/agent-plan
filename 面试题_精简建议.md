# 面试题精简建议（5-7 年 Java 工程师）

> 目标：删减「背题就能过、难以区分候选人真实水平」的题目，保留/强化「能暴露问题、串联能力」的题目。

---

## 一、建议删减（区分度低 / 过于基础）

| 题目 | 理由 |
|------|------|
| **foreach 循环中能否移除元素** | 1-2 年水平，背题即答，不考察思维 |
| **@Autowired 和 @Resource 的区别** | 纯记忆（按名称/按类型），查文档即可，实战价值低 |
| **反射机制，几种方式** | Class.forName / .class / getClass，太基础 |
| **枚举的实现原理** | 原文档未展开，且业务中接触少，偏冷门 |
| **Filter 与 Interceptor 的区别** | 表格背诵型，实际用到时查即可 |
| **LIKE %abc% 能否使用索引** | 一句话能答完，无追问空间 |
| **IN (1,2,3) 能否使用索引** | 同上 |
| **InnoDB vs MyISAM** | 生产 99% 用 InnoDB，MyISAM 几乎不用，对比意义弱 |
| **Docker 复制日志文件** | 纯命令记忆（docker cp），不考察问题解决能力 |
| **JVM 内存模型与各区域作用** | 程序计数器/栈/堆/方法区，人人能背，无区分度 |

## 三、建议精简（大题改追问，或弱化）

| 题目 | 建议 |
|------|------|
| **volatile 是什么** | 保留，但**直接当追问**：volatile 能替代 synchronized 吗？i++ 用 volatile 是否线程安全？定义可省略 |
| **Eureka vs Nacos** | 表格缺失时价值有限；可改为「服务注册发现选型考虑什么？」更开放 |
| **除了 @Autowired 还有什么方法获取 Bean** | 改为场景追问：「如果要在非 Spring 管理的类里拿 Bean，怎么做？」 |
| **Spring Boot 配置优先级** | 改为「多环境配置怎么管理？优先级遇到过坑吗？」 |
| **BIO/NIO/AIO** | 精简：只问 NIO 与 Netty 的关系，BIO/AIO 可省略 |
| **零拷贝** | 非文件/消息中间件方向可删；或改为场景题追问 |

## 四、建议保留且加强追问的题目

| 题目 | 加强方向 |
|------|----------|
| **HashMap / ConcurrentHashMap** | 追问：为什么 8 转红黑树？hash 扰动？JDK7 死循环如何复现？ |
| **synchronized** | 追问：锁升级触发条件？JDK15 偏向锁变化？与 ReentrantLock 选型场景？ |
| **循环依赖** | 追问：构造器为什么不行？三级缓存每层存什么？ |
| **@Transactional 失效** | 追问：同类调用为什么失效？画一下代理调用链 |
| **MVCC / 幻读** | 追问：快照读 vs 当前读对幻读的差异？ |
| **Redis 穿透/雪崩/击穿** | 追问：为什么击穿用互斥锁、穿透用布隆？能混淆说明没理解 |
| **线程池** | 追问：队列满后 CallerRunsPolicy 会怎样？为什么 IO 密集要更多线程？ |

## 六、核心原则

1. **能背的少问**：定义、对比表格类，查文档即得，不考察深度。
2. **能串联的多问**：场景题、排查题、设计题，看能否把多技术栈串起来。
3. **有坑的深问**：事务失效、缓存一致性、主从延迟，有实际踩坑经验才能答好。
4. **追问优先**：基础题可以保留，但**直接当追问**用，不必让候选人完整背定义。
